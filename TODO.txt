Jerarquía de Casillas, una abstracta de Casilla con métodos y atributos comunes a todas ellas. Segundo nivel clases finales
Propiedad, Accion, Impuesto, Grupo y Especial, donde Grupo tengan relación de composición con clase Propiedad, un Grupo tiene
una lista de Propiedades. Tercer nivel de tres tipos de propiedades: Solar, Servicio y Transporte; y los tipos de acciones
Suerte, CajaComunidad y Parking.

Casilla debe tener boolean estaAvatar( ) frecuenciaVisita String toString( ). Estas funciones ya las tenemos parcialmente, tenemos un
getAvatares() el cual devuelve una lista de Avatares, al cual podemos decir getLugar(). frecuanciaVisita será un wrapper del Singleton
de StatsTracker en esa Casilla. toString otro wrapper a las subclases que tienen Descripción.

En Propiedad se deberán de definir, al menos, los siguientes métodos:
   • boolean perteneceAJugador(Jugador jugador)
   • boolean estaHipotecada( )
   • void hipotecar( )
   • abstract boolear alquiler( )
   • abstract float valor( )
   • void comprar(Jugador jugador)

Jerarquía de los Edificios, clases abstracta llamada Edificios con métodos y atributos comunes. Segundo nivel con
Casa, Hotel, Piscina y PistaDeporte.

Jerarquía de Cartas, raíz de clase Carta con métodos y atributos comunes, y un segundo nivel compuesto de Suerte y CajaComunidad.
Carta debe tener void accion( )

clase Juego tendra un atributo estático del tipo ConsolaNormal para que los métodos de imprimir y leer se puedan invocar
desde cualquier clase del programa sin tener que instanciar continuamente dicha clase
